# 🐛 Quick Debugging Reference

## 3 Ways to Debug (Pick One)

### ⚡ EASIEST: Simple Interactive Menu
```bash
python simple_debug.py
```
Menu-driven debugging with step-by-step walkthroughs.

### 🔧 VS CODE: Visual Debugger
```
1. Press ⇧⌘D (or View → Debug)
2. Select debug config from dropdown
3. Click F5 or "Start Debugging"
4. Click line number to add breakpoint
5. Use F10 (step over), F11 (step into), F5 (continue)
```

### 💻 TERMINAL: Command Line Debugger
```bash
python debug_api.py
# or
python debug_transcript_extractor.py
```

### 🪲 "I found a bug and want to step through it"
```
1. Open VS Code
2. Open the file with the bug
3. Click on the line number to add breakpoint (red dot)
4. Press F5 to start debugging
5. Press F10 to step line-by-line
6. Use Variables panel to inspect state
```

## Common pdb Commands (When Debugger Pauses)

```
n          Next line (step over)
s          Step into function
c          Continue
p var      Print variable
pp dict    Pretty-print
l          List code
w          Show where you are
h          Help
q          Quit

# Examples:
(Pdb) p result
(Pdb) pp result['summary']
(Pdb) p len(segments)
```

## Useful Debug Patterns

### Print Structure
```python
(Pdb) import json
(Pdb) print(json.dumps(result, indent=2, default=str)[:500])
```

### Check Types
```python
(Pdb) p type(result)
(Pdb) p type(result['transcript'])
(Pdb) p hasattr(result, '__dict__')  # Is it an object?
```

### Inspect Collections
```python
(Pdb) p len(segments)
(Pdb) p segments[0]
(Pdb) p [s.text for s in segments[:3]]
```

### Check Errors
```python
(Pdb) import traceback
(Pdb) traceback.print_exc()
```
